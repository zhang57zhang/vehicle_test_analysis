# 车载控制器测试数据分析与测试报告编写系统 - 需求文档

## 1. 项目概述

### 1.1 项目目标
建立本地部署的测试数据分析与报告生成系统，支持MIL/HIL/DVP/整车测试的数据分析与报告生成。

### 1.2 系统定位
- 本地部署，数据安全可控
- 多用户、多项目管理
- 支持离线运行

---

## 2. 运行环境

### 2.1 目标平台
| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10 / Windows 11 |
| 其他平台 | 不支持 Linux |
| 安装包大小 | 无限制 |

### 2.2 部署模式
- 完全本地部署
- 无云端依赖
- 支持离线安装和运行

---

## 3. 数据规模与特性

### 3.1 数据规模
| 维度 | 要求 |
|------|------|
| 单次测试数据 | 2KB ~ 500MB |
| 数据源数量 | 单次测试可能有多种数据源 |
| 时间同步精度 | 10ms，若CAN通信周期更小则以最小周期为精度 |

### 3.2 时间同步要求
- 多数据源时间对齐
- 精度：10ms 或更高（取决于最小采样周期）
- 支持不同时间戳格式的统一转换

## 6. 测试用例与指标定义

### 6.1 测试用例来源
| 阶段 | 来源 | 优先级 |
|------|------|--------|
| 当前版本 | Excel文件 | P0 |
| 后续迭代 | 测试管理系统 | P1 |

### 6.2 指标定义来源
1. **从测试用例提取**
   - 解析Excel中的测试用例
   - 自动识别关注信号和判定标准

2. **独立指标定义文件**
   - 针对不同试验类型
   - YAML/JSON格式配置
   - 可复用的指标模板

### 6.3 指标类型
- 单信号值
- 信号计算值
- 时域指标（响应时间、超调量等）
- 统计指标（均值、标准差、百分位数等）
- 自定义公式

### 6.4 计算公式支持
| 计算类型 | 说明 |
|----------|------|
| 基本运算 | 加减乘除、幂运算 |
| 数据组合 | 多信号组合计算 |
| 微分 | 信号变化率计算 |
| 积分 | 时间积分、能量计算 |
| 滤波 | 低通、高通、带通滤波 |
| 统计 | 均值、方差、百分位数 |
| 时间窗口 | 滑动窗口计算 |

## 7. 分析功能

### 7.1 功能测试分析
- 信号值范围检查
- 信号状态跳转验证
- 信号时序关系验证
- 逻辑一致性检查

### 7.2 性能测试分析
- 响应时间分析
- 统计分布计算
- 趋势分析
- 性能瓶颈识别

### 7.3 数据确认与调整
- 分析结果支持工程师确认
- 支持手动调整判定结果
- 调整记录可追溯

---

## 8. 报告系统

### 8.1 报告输出格式
- Word (.docx)
- PDF (.pdf)

### 8.2 双报告机制
| 报告类型 | 用途 | 内容 |
|----------|------|------|
| 正式报告 | 对外交付、归档 | 测试结论、指标结果、图表 |
| 审核报告 | 内部审核、问题追溯 | 正式报告 + 数据溯源标注 |

### 8.3 数据溯源标注内容
- 数据来源（文件名、路径、大小、采集时间）
- 信号信息（信号名、报文ID、DBC文件、单位）
- 数据范围（时间范围、数据点数、原始值范围）
- 计算过程（计算方法、公式、中间结果）

### 8.4 报告模板
- 按测试阶段分类：MIL、HIL、DVP、整车
- 按测试类型分类：功能、性能、安全
- 支持车企模板导入与适配

---

## 9. 工具集成

### 9.1 不支持的集成
- 不与CANoe、INCA等工具直接联动
- 不需要实时数据采集

### 9.2 支持的数据来源
- 从各工具导出的测试结果数据
- 从各工具导出的测试日志
- 从各工具导出的测试用例（Excel格式）

### 9.3 解析库来源
- 优先使用Python开源库
- 自研特殊格式解析器

## 10. 技术架构

### 10.1 技术栈
```
后端语言:     Python 3.10+
数据处理:     Pandas, NumPy, SciPy
CAN解析:      cantools, python-can
MDF解析:      asammdf
数据库:       SQLite
报告生成:     python-docx, reportlab
桌面界面:     PyQt6 / PySide6
```
